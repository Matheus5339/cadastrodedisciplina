## 1. objetivo
 
   fazer um controle das disciplinas cursadas ao longo do curso de engenharia da computação na ucp.
   o sistema deve permitir que se gerencie as disciplinas cursadas por um único aluno.

## 2. requisitos
   
   . inserir, remover, alterar, consultar uma disciplina
   . listar e filtrar disciplinas, por nome, nome professor, semestre e/ou ano
   . mostrar o cr (coeficiente de rendimento)
   . ter controle de acesso, cada usuário controla somente as suas disciplinas, não pode ver os dados
     dos demais usuários.
   . um usuário não cadastrado, pode se cadastrar normalmente, não precisa checar se o email existe.
     mas deve ser um email valido.
   . caso exista um arquivo discipolinas.csv, em um diretório especifico, esse deve ser importado para o banco de dados, atualizando a tabela disciplinas. (isso so deve ser feito no inicio da aplicação)
    
   
## 3. modelo de dados

   disciplina
     id:inteiro
     código: string
     nome: string
     professor: string
     periodo : inteiro
     creditos : inteiro
 
   aluno
      id : inteiro
      rgu:string
      cpf: string
      email: string
      nome : string
      foto : imagem
      
   historico
      id:inteiro
      idAluno:inteiro
      idDisciplina:inteiro
      ano:inteiro
      semestre:inteiro
      periodo: inteiro
      mediaFinal: real
